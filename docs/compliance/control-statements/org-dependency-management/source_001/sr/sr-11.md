# sr-11 - \[SR\] Component Authenticity

## Control Statement

Develop and implement anti-counterfeit policies and procedures that include the means to detect and prevent counterfeit components from entering the system.

## What is the solution and how is it implemented?

### org-dependency-management

SHA pinning prevents tag-hijacking attacks where an attacker force-pushes a malicious commit to an existing tag. The SHA-256 hash guarantees the Action code matches what was originally reviewed.

Centralized SAST analysis further detects ref-confusion and impostor commit risks — cases where ambiguous git references could resolve to unintended (potentially malicious) code. Together, SHA pinning and SAST scanning prevent counterfeit or tampered dependencies from entering the CI/CD pipeline.
